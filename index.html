git提交代码到coding的一些分享

coding是国内的一个代码托管平台，是一个高效的研发流程与管理平台，它让多个同事同时维护一个项目更加方便了。
但是我在coding的道路上踩过很多坑。。。

【辅助网址：】
http://www.liaoxuefeng.com/wiki/0013739516305929606dd18361248578c67b8067c8c017b000/00137621280731812dec22ecc9b44f4b2ca1c680f181a5b000

=========================================================================

【克隆远程版本库（注意版本库的地址）】

git clone https://git.coding.net/liu2017/DDdingshui.git

=========================================================================

【拉取代码】

git pull

=========================================================================

【提交代码】

1.把文件加入到暂存区（修改、新增的文件）
git add . //全部
git add file.php //file.php文件

2.提交到本地版本库（注意版本说明必填）
git commit -m ‘版本说明’

3.推送至远程版本库
git push origin master

==========================================================================

【解决冲突】

1.先拉取远程版本库代码
1）git pull //拉取远程版本库代码，自动合并到本地代码

2) git fetch origin master:tmp
git diff tmp
git merge tmp //拉取远程版本库代码，不会自动合并（理论上此方法更安全）

2.git status //查看版本状态，查看冲突原因

3.打开冲突的文件，找到冲突的位置，如下：（以=====隔开，上面的是本地代码，下面的是服务器上的代码）
‘<<<<<<<’ HEAD
我是本地代码（2）
“=======”
我是服务器上的版本（1）
“>>>>>>>” 19ff362904a580b17634a007beb59b4ae9f7cc93

4.手动处理冲突，把<<<<<<<< >>>>>>>>>符号删掉

5.重新执行【提交代码】中的步骤

注意：解决冲突中的步骤中１中的１２步只需要选择其中一种操作

===========================================================================

【合并冲突】

git merge //合并冲突并没有解决冲突，而是把本地和服务器上的版本合并，然后可以推送到远程版本库

==========================================================================

【注意】

2.有时提交的时候没发现什么错误，却提交不上去，此时，查看该文件夹下是否有git的忽略文件，如果有，删除即可。

==========================================================================

rm -rf .git //删除本地分支 （本地分之乱码的情况）
git init //重新初始化git

==========================================================================

【本地创建分支提交coding】

git init //初始化一个分支

git remote add origin(本地分支的名字) 地址（coding上的地址）

git add -A //提交全部

git commit -m ‘注释’

git push origin(本地分支的名字) master:分支名 //把本地分支提交到coding上

删除分支 git push origin(本地分支的名字) -d front （不可以删除分支，会导致历史记录被清空）

切换分支：git checkout qianduan

查看有什么修改的代码：git status

==========================================================================

【clone分支】

克隆github上的仓库到本地，默认会把仓库的所有内容clone到本地。

cd delun（进入delun文件夹） git branch -r （查看分支所有分支名）

这时用 git checkout front(分支名) 操作就把远程分支取到本地。

这时再用不带-r的git branch命令就能看到刚才操作的分支名了。

==========================================================================

ls -a (查看当前有没有git文件)

git remote //查看本地分支

其实程序员是这样的

你眼里的程序员是怎样的呢？你会给他们贴上什么标签呢？

2018年3月，中国青年报社做了一个调查，显示86%的受访者认为职业标签化现象普遍。调查中，程序员群体自然不能跳过，提到程序员，自我，宅，单身，夜猫子，屌丝，格子衫 …… 人们都会想到这些。作为高智商，高收入的精英群体，但经常被叫做IT民工，码农，程序猿，被贴各种各样的标签，那他们到底是怎么样的呢？我想说，有些程序员并不符合这些人设，至少我身边的程序员就不是。

程序员都是单身吗？开什么玩笑，程序员的女朋友都很漂亮呢，越厉害的程序员女朋友也越漂亮。现在程序员可是丈母娘最爱的女婿，人多金又老实，有着强大的大脑，遇事沉着冷静，就像他们遇到bug一样，需要头脑冷静，从容的去对待。

程序员宅吗？不，我们公司程序员平日爱好，也是五花八门，有爬山、攀岩这种动态的，也有摄影、看剧这种静态的，她们也是经常三天一小聚，五天一大聚的，因为他们双休有时间呀，并不是所有程序员都要经常加班的，他们只要在规定的时间完成项目就可以了，时间自由分配。

程序员都是格子衫吗？不，我们公司程序员穿的衣服都是各式各样的搭配，下雨天都不会想着要穿拖鞋，每天过来发型也是精心打理过的样子。聚会时打闹的样子也都要帅帅美美的，发型不能乱，衣服要干干净净的。就这样，你还要说他们是格子衫的屌丝吗？

程序员都是夜猫子吗？不可能的，我们公司的程序员都是每天用保温瓶来泡枸杞，玻璃瓶来泡菊花茶喝的养身T。如果不是刚好碰到产品一定要今天上线和之前的准备不够充分，怎么会加班呢。你不是程序员，难道你就没有加过一次班，睡过一次晚觉吗，可能吗？

所以说，程序员的生活是丰富的，充实的，好玩的，并且不是你想当就可以当的。在公司，可以跟老板聊产品，跟行政小姐姐聊这个星期的团建去哪玩，这个月的聚餐去哪happy，不用打卡，上班自由，水果，饮料，零食等着你，你不喜欢吗？

成为了一个程序员，我也很意外

记得小学时候的梦想是想当个老师，当时写的作文，在讲台上字字句句读的场景，好像历历在目。转眼间，已经过去了那么多年。在高考后报考志愿的时候，我是迷茫的，也没有人给我建议，当时的自己也把小时候的梦想抛掷脑后了，所以当时是看着哪个专业比较舒服就报考哪个专业，后来就被广东理工职业学院的计算机应用专业录取了。

在大学的前两年，我基本也是玩过来的，在保证不挂科的情况下，怎么开心，怎么玩。总感觉自己上了个假大学，不挂科，你确定你是在上大学吗？在最后一年里，学校开了个前端班和Java班，基于之前在学网页制作的时候挺感兴趣的，所以就去报名了，给自己一个方向。

在前端班学习的半年期间，我发现自己越来越喜欢这门技术了，觉得自己设计出一个网站再用代码敲出来，很有成就感；每次遇到bug，到再修复的过程,每次自己都觉得很有成就感。在学校学的都是有限的，半年之后出来实习了，实习的时候遇到过两个组员，他们很耐心的去引导我，也使我在前端的领域里接触到更多，了解更多，也更加喜欢上这份工作了。
